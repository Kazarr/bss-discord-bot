## Overview

**bss-discord-bot** is a standalone, stateless Node.js Discord bot for collecting game feedback on "By Sword and Seal." The bot uses Discord's slash commands to initiate private threaded conversations, Claude API for multi-turn interaction and semantic analysis, and GitHub API to create/attach issues from user feedback.

**Key property:** Greenfield project (new repository), not part of the game monorepo. Runs as a separate process with independent CI/CD.

## Service Architecture

### `src/services/github.service.ts`

**Purpose:** Octokit wrapper with caching and pagination.

**Public API:**
```typescript
class GitHubService {
  async fetchAllIssues(): Promise<GitHubIssue[]>;
  async createIssue(data: IssueData): Promise<GitHubIssue>;
  async addComment(issueNumber: number, body: string): Promise<void>;
  async refreshCache(): Promise<void>;
}
```

**Details:**
- Manages in-memory cache of issues with 15-minute TTL
- Uses `octokit.paginate()` for automatic pagination of large result sets
- Error handling: 404 (repo not found), 403 (rate limit), network errors
- Cache invalidated after `createIssue()` to ensure next fetch gets fresh list
- All GitHub operations require valid authentication token and correct repo

**Rate limiting:** 5,000 requests/hour with personal access token (sufficient for v1).

### `src/services/claude.service.ts`

**Purpose:** Anthropic SDK wrapper for conversation, summarization, scope filtering, and semantic similarity.

**Public API:**
```typescript
class ClaudeService {
  async chat(conversationMessages: Message[], userMessage: string): Promise<string>;
  async summarize(conversationMessages: Message[]): Promise<string>;
  async isGameRelated(message: string): Promise<boolean>;
  async findSimilarIssue(summary: string, issues: GitHubIssue[]): Promise<SimilarityResult>;
}
```

**System Prompts:**
- **Conversation:** "You are a game feedback collector for By Sword and Seal. Communicate in **Slovak**. Ask clarifying questions about game mechanics, bugs, and UI/UX. Reject completely off-topic messages."
- **Similarity:** "Compare this summary against the following GitHub issues. If one is clearly about the same topic, return its issue number. If no match, return 'none'. Confidence: high/medium/low."
- **Scope filter:** Run on first user message to reject off-topic feedback immediately.

**Details:**
- Conversation history is passed in (not stored) — external caller manages history
- Claude model configurable via `CLAUDE_MODEL` env var
- Summarization generates a concise issue title and description from multi-turn conversation
- Similarity comparison sends all issue titles/bodies to Claude for semantic analysis (no embeddings API)

## Handler Architecture

### `src/handlers/message.handler.ts`

**Purpose:** State machine that routes messages through the conversation flow.

**State Machine Diagram:**
```
START: User invokes /issue command
  ↓
[collecting] ← user message → ClaudeService.chat() → bot reply
  ↓ (when Claude determines enough info)
[summarizing] → ClaudeService.summarize() → bot shows summary in Slovak
  ↓
[confirming] ← "Vytvoriť?" prompt shown
  ↓
User responds:
  "áno" / "yes" → [creating] → GitHubService.createIssue() → close thread → [done]
  "nie" / "no" → [refining] → ask "refine or cancel?" 
    "refine" → [collecting] ← (loop back to collecting more feedback)
    "cancel" → close thread → [done]
```

**Public API:**
```typescript
class MessageHandler {
  constructor(
    claudeService: ClaudeService,
    githubService: GitHubService,
    threadService: ThreadService
  );

  async handleMessage(message: Message): Promise<void>;
}
```

**Details:**
- Maintains `Map<string, ConversationState>` for in-progress conversations
- On message in a tracked thread: load state, route to handler, update state
- On GitHub create: format issue with Discord user info + conversation summary
- On issue attach (duplicate found): add comment linking back to conversation
- Cleanup: remove state entry when thread reaches 'done'
- Error handling: service failures, Discord permission errors, GitHub API errors

## Conversation Flow (Detailed)

### Phase 1: Collecting (User Provides Feedback)

1. User: `/issue` → bot creates private thread with welcome message
2. User sends message in thread describing an issue
3. Bot runs `ClaudeService.isGameRelated()` on first message:
   - If **off-topic:** Reply "To nie je o hre. Skus to znova.", close thread, mark done
   - If **on-topic:** Continue to conversation
4. Bot calls `ClaudeService.chat()` with conversation history + user message
5. Bot sends Claude response in thread
6. Loop: Continue until Claude determines enough info (heuristic: ~4-5 exchanges or explicit "ok, summary time")

### Phase 2: Summarizing (Claude Generates Issue Summary)

1. Bot calls `ClaudeService.summarize()` on entire conversation
2. Claude returns structured summary:
   ```
   Title: [one-line issue title]
   
   Description: [multi-paragraph issue description]
   ```
3. Bot sends summary in thread, transitions to 'confirming' phase

### Phase 3: Confirming (User Approves or Refines)

1. Bot displays summary with Slovak prompt: **"Vytvoriť?"** (Create?)
2. User reacts or responds:
   - **Approves** (responds "áno" / "yes" / positive emoji): Transition to creating
   - **Declines** (responds "nie" / "no" / negative emoji): Ask "Want to refine or cancel?"
     - "refine" → loop back to [collecting]
     - "cancel" → close thread, mark done

### Phase 4: Creating Issue

1. Bot calls `ClaudeService.findSimilarIssue()` with summary + all existing issues
2. Result:
   - **No similar issue found:** Call `GitHubService.createIssue()` with formatted title + body
     - Body format:
       ```
       **Discord User:** {username}#{discriminator}
       **Reported:** {timestamp}
       
       {issue description from summary}
       
       **Thread:** {link to Discord thread}
       ```
     - Reply with "✅ Issue created: {GitHub issue link}"
   - **Similar issue found:** Call `GitHubService.addComment()` on existing issue
     - Comment links back to original report and provides context
     - Reply with "🔗 Attached to existing issue: {GitHub issue link}"
3. Close thread: `ThreadService.closeThread()` (archive + lock)
4. Remove state entry, mark done

## External API Integrations

### Discord API (discord.js)

**Intents required:**
- `Guilds` — Access guild and channel structure
- `GuildMessages` — Receive messages in threads
- `MessageContent` — Read message content (PRIVILEGED — must enable in Dev Portal)

**Permissions required (bot role):**
- `SendMessages` — Send messages in channels/threads
- `CreatePrivateThreads` — Create private threads
- `SendMessagesInThreads` — Send messages inside threads
- `ManageThreads` — Archive/close threads, manage settings
- `ReadMessageHistory` — Read conversation history for scope filtering

### Anthropic API (@anthropic-ai/sdk)

**Endpoint:** `POST /messages` (Messages API v1)

**Usage:**
- Multi-turn conversation with system prompt
- Stateless — conversation history passed in each request
- Model configurable (default: `claude-sonnet-4-20250514`)
- Max tokens: 1024 per response
- Cost: Standard Anthropic pricing

**No embeddings API** — Semantic similarity done via prompt-based classification.

### GitHub API (@octokit/rest)

**Endpoints used:**
- `GET /repos/{owner}/{repo}/issues` (with pagination)
- `POST /repos/{owner}/{repo}/issues` (create new)
- `POST /repos/{owner}/{repo}/issues/{number}/comments` (add comment)

**Authentication:** Personal Access Token (5,000 requests/hour limit)

**Target repository:** Configurable via `GITHUB_OWNER` and `GITHUB_REPO` env vars.

## In-Memory State Management

**Key-value store:** `Map<threadId, ConversationState>`

```typescript
const activeConversations = new Map<string, ConversationState>();

// On /issue command:
activeConversations.set(threadId, {
  threadId,
  userId,
  messages: [],
  phase: 'collecting',
});

// On each user message:
const state = activeConversations.get(threadId);
state.messages.push({ role: 'user', content: userMessage });
// ... update phase, summary, etc.

// On thread close or timeout:
activeConversations.delete(threadId);
```

**Limitations:**
- Lost on bot process restart (accepted for v1)
- Threads auto-archive after 60 minutes of inactivity (cleanup mechanism)
- No persistence to database
