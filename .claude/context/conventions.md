### Testing Strategy

- **Unit tests only** — no integration tests with real APIs for v1
- **Mock all external services:** Discord, Anthropic, GitHub, filesystem
- **Module-level mocking:** Use `vi.mock('discord.js')` pattern
- **ESM Mocking Pattern (CRITICAL - Lesson #14):** For ESM projects, ALL external API mocks MUST use `vi.hoisted()` pattern:
  ```typescript
  const { mockFn } = vi.hoisted(() => ({ mockFn: vi.fn() }));
  vi.mock('@package/module', () => ({ default: class { method = mockFn } }));
  ```
  This is required because ESM hoisting prevents module-scope `vi.fn()` without `vi.hoisted()`.
- **Test coverage targets:**
  - Config validation (missing vars, defaults)
  - All GitHub service methods (fetch, create, comment, cache)
  - All Claude service methods (chat, summarize, scope filter, similarity)
  - All thread service methods (create, message, archive)
  - Full state machine branches (including decline, cancel, off-topic paths)
  - Error handling (404s, 403s, network errors)

## Conversation Flow & State Machine

**Conversation phases:**
1. **collecting** — User provides feedback, bot asks clarifying questions via Claude
2. **summarizing** — Claude generates issue summary from conversation
3. **confirming** — Bot shows summary in Slovak ("Vytvoriť?"), user confirms/declines
4. **done** — Issue created/attached and thread closed

**Prompts (language requirement):**
- System prompt for conversation: Communicate in **Slovak**, accept game mechanics/bugs/UI/UX, reject off-topic
- Similarity prompt: Compare summary against existing GitHub issues, return best match or "none"
- Confirmation prompt: Present summary in Slovak with "Vytvoriť?" (Create?) button/reaction

## GitHub Integration

| Operation | Method | Rate Limit Impact | Cache Behavior |
|-----------|--------|-------------------|-----------------|
| **Fetch all issues** | `octokit.paginate()` on `issues.listForRepo` | ~0.01 req per issue | Cache with 15-min TTL or manual refresh |
| **Create issue** | `issues.create()` with title + body | 1 req | Invalidate cache immediately |
| **Add comment** | `issues.createComment()` on issue number | 1 req | No cache impact |
| **Refresh cache** | Manual call to `refreshCache()` | Triggers full fetch | Reset TTL |

## Known Limitations & Gotchas

1. **Forum channels cannot have private threads** — Use regular text channels instead (codebase-map.md CRITICAL FINDING)
2. **MessageContent is a privileged intent** — Must be manually enabled in Discord Developer Portal (codebase-map.md)
3. **Bot restart loses conversation state** — In-memory storage only for v1; accepted limitation (codebase-map.md)
4. **No embeddings API from Anthropic** — Use prompt-based semantic similarity instead (codebase-map.md)
5. **Target GitHub repo may not exist** — Configurable via env vars to handle mismatch (codebase-map.md CRITICAL FINDING)
6. **Prompt-based similarity doesn't scale past ~200 issues** — v2 should add embeddings API (codebase-map.md)
7. **Rate limiting:** GitHub allows 5,000 requests/hour with PAT (sufficient for v1 low volume) (codebase-map.md)
